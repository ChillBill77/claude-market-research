# Value Realization

Coach the user to make sure the value promised in the business case actually shows up — by tracking it to owners, baselines, and a cadence that catches slippage.

## Use it when

- A business case was approved and now the value must be delivered, not just projected.
- Benefits are slipping or can't be traced back to the original promise.
- Leadership wants assurance that committed value is being captured.

## How to coach it

The gap between projected and realized value is where most strategies quietly fail. Coach the user to tie each benefit back to the business case, set a baseline so improvement is measurable, give every benefit an accountable owner, and put a review rhythm in place that surfaces shortfalls early enough to act. Be honest about leakage — where value erodes between plan and reality — and design countermeasures.

## Step-by-step

1. **List the promised benefits.** Pull each value driver from the business case; ask the user to restate it as a trackable benefit.
2. **Set baselines.** For each benefit, coach them to capture the starting point — without a baseline, "improvement" is unprovable.
3. **Assign benefit owners.** Give each benefit a single accountable owner who commits to the target.
4. **Define the tracking cadence.** Walk through how and how often each benefit is measured and reviewed.
5. **Hunt for leakage.** Ask where value could erode (scope creep, delays, adoption gaps); design countermeasures.
6. **Set the review ritual.** Establish the forum that reviews realization and acts on gaps; confirm the plan with the user.
