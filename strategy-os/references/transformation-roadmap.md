# Transformation Roadmap

Coach the user to lay the prioritized initiatives onto a phased timeline with milestones, owners, and dependencies — a plan people can actually execute against.

## Use it when

- Priorities are set and now need a sequenced, time-bound execution plan.
- Multiple workstreams must be coordinated across months/quarters.
- Leadership needs to see milestones, owners, and when value lands.

## How to coach it

Build the roadmap in waves, not as a Gantt chart of everything at once. Coach the user to define a small number of phases, each with a clear theme and an outcome that proves progress. Insist every milestone has a single owner and a date, and that early phases create the foundations later ones depend on. Sequence for early, visible wins to build momentum, while protecting the critical path. Make risks to the timeline explicit.

## Step-by-step

1. **Set the phases.** Ask the user to group the work into 2–4 waves, each with a theme; help them name the outcome that signals each phase is done.
2. **Place initiatives into phases.** Walk through the prioritized list, slotting each by dependency and capacity; resolve conflicts live.
3. **Define milestones.** For each phase, coach them to set a few concrete, dated milestones — assign one owner each.
4. **Check the critical path.** Ask what would delay everything if it slipped; make sure those items are protected and sequenced first.
5. **Mark where value lands.** Help them show when benefits actually start accruing, not just when work finishes.
6. **Surface timeline risks.** Ask what could blow the schedule and what the mitigation or buffer is; confirm the whole roadmap with the user.

## Deliverable

```markdown
# Transformation Roadmap — [strategy]

## Phases
| Phase | Theme | "Done when…" outcome |
|---|---|---|

## Milestones
| Phase | Milestone | Owner | Date | Depends on |
|---|---|---|---|---|

## Critical path
[The items that gate everything; sequenced first]

## Value timing
[When benefits begin to land, by phase]

## Timeline risks
| Risk to schedule | Mitigation / buffer |
|---|---|
```
