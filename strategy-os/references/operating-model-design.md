# Operating Model Design

Coach the user to design how the organization must work to actually deliver the chosen strategy — capabilities, structure, decision rights, and ways of working.

## Use it when

- A strategy is chosen and the question becomes "how do we organize to deliver it?"
- The current structure or decision-making is blocking the strategy.
- Re-org, scale-up, integration, or a new capability build is on the table.

## How to coach it

Lead the user one element at a time; don't hand them a finished org chart. Start from the strategy, not from boxes-and-lines — the operating model exists to deliver specific outcomes, so always tie each choice back to "what does the strategy need here?" Keep them honest about the gap between today's model and what's required. Surface trade-offs (e.g. centralize for consistency vs. decentralize for speed) and make them choose deliberately rather than defaulting.

## Step-by-step

1. **Anchor on what the strategy demands.** Explain that the model serves the strategy; ask the user which few capabilities the strategy makes mission-critical. Help them narrow to the vital ones.
2. **Assess the current model.** Ask how work, decisions, and accountability flow today for those capabilities. Coach them to name where it breaks.
3. **Design the target capabilities.** For each critical capability, ask what "good" looks like and what's missing (people, process, tech, data). Help them be concrete.
4. **Set structure and decision rights.** Walk them through who owns what and who decides what. Use a simple accountability lens (e.g. who's responsible vs. who approves) and resolve overlaps live.
5. **Name the ways of working.** Ask what rhythms, metrics, and interfaces connect the parts. Help them avoid orphaned hand-offs.
6. **Confirm the gap and the moves.** Summarize current → target and the changes required; confirm with the user before closing.

## Deliverable

```markdown
# Operating Model — [strategy]

## What the strategy demands
[The few mission-critical capabilities]

## Capability map (current → target)
| Capability | Today | Target | Gap |
|---|---|---|---|

## Structure & decision rights
| Area | Owner (responsible) | Decider (approves) | Notes |
|---|---|---|---|

## Ways of working
[Key rhythms, metrics, interfaces between units]

## Changes required
- [The deliberate moves, with the trade-off each one accepts]
```
