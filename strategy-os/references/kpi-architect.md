# KPI Architect

Coach the user to design a small set of metrics that actually track whether the strategy is working — leading and lagging, with targets and owners.

## Use it when

- A strategy is committed and needs measurement that drives behavior.
- The current dashboard is a wall of numbers nobody acts on.
- Leadership needs to know, early, whether the plan is on track.

## How to coach it

Fight metric sprawl. Coach the user toward the few KPIs that connect directly to the strategy's success, and balance lagging outcomes (did it work?) with leading indicators (will it work?). Every KPI needs a clear definition, a target, and an owner — an unowned metric is decoration. Watch for metrics that can be gamed or that drive the wrong behavior, and design around them.

## Step-by-step

1. **Restate what success means.** Ask the user what outcome the strategy must produce; help them phrase it as something measurable.
2. **Pick lagging outcome metrics.** Coach them to choose the few that prove success after the fact — and only those.
3. **Add leading indicators.** Ask what early signals predict those outcomes; help them find metrics that move sooner.
4. **Define each KPI precisely.** Walk through definition, data source, and frequency so two people would compute it the same way.
5. **Set targets and owners.** For each KPI, set a target and a single owner; challenge vanity targets.
6. **Pressure-test for gaming.** Ask how each metric could be hit while missing the intent; adjust or pair metrics to prevent it. Confirm the final set is small and actionable.

## Deliverable

```markdown
# KPI Set — [strategy]

## Success definition
[What "working" means, measurably]

## KPIs
| KPI | Lead/Lag | Definition & source | Target | Owner |
|---|---|---|---|---|

## Anti-gaming notes
[How each at-risk metric is protected from perverse behavior]
```
